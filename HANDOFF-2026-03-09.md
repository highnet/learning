# Handoff — March 9, 2026

## Goal
Refactor the monorepo into a picture-led blog with:
- authoring in Payload CMS
- public blog in `apps/web`
- Redis for cache/session/rate-limit foundations
- RabbitMQ for content/media events and async jobs
- Drizzle read-model projection synced from Payload

## Decisions already made
- Payload is the write source of truth.
- Drizzle is being used as the app-facing projection/read model.
- Public blog lives in `apps/web`.
- Redis is intended for blog caching and related web concerns.
- RabbitMQ is intended for content/media events and async jobs.
- Initial blog scope includes:
  - featured images
  - inline rich-text images
  - tags/categories
  - SEO fields
  - draft/published workflow
  - galleries
